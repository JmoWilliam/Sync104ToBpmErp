# Sync104ToBpmErp 部署說明

## 部署前準備

### 1. 確認客戶環境
- [ ] Windows Server (建議 Windows Server 2016 以上)
- [ ] .NET 8.0 Runtime 已安裝
- [ ] 可連線到 HR API
- [ ] 可連線到 BPM 資料庫 (MS-SQL 2008)
- [ ] 可連線到 ERP 資料庫 (Oracle 11g)

### 2. 取得必要資訊
向客戶索取以下資訊：

#### HR API 資訊
- API Base URL
- API Key
- 各端點路徑 (Auth, Employee, Department, Hierarchy)

#### BPM 資料庫 (MS-SQL 2008)
- 伺服器位址
- 資料庫名稱
- 使用者帳號/密碼

#### ERP 資料庫 (Oracle 11g)
- 伺服器位址
- Service Name / SID
- 使用者帳號/密碼

## 部署步驟

### 步驟 1: 建立資料表

在客戶資料庫執行 SQL 腳本：

```bash
# BPM 資料庫 (MS-SQL)
執行 SQL/BPM_Schema.sql

# ERP 資料庫 (Oracle)
執行 SQL/ERP_Schema.sql
```

### 步驟 2: 修改設定檔

編輯 `appsettings.json`，填入客戶環境的連線資訊：

```json
{
  "HRApi": {
    "BaseUrl": "https://客戶-HR-API-網址",
    "AuthEndpoint": "/api/auth/token",
    "EmployeeEndpoint": "/api/employees",
    "DepartmentEndpoint": "/api/departments",
    "HierarchyEndpoint": "/api/hierarchy",
    "ApiKey": "客戶提供的API-Key"
  },
  "BpmDatabase": {
    "ConnectionString": "Server=客戶SQL伺服器;Database=BPMDB;User Id=帳號;Password=密碼;TrustServerCertificate=True;",
    "Provider": "MSSQL"
  },
  "ErpDatabase": {
    "ConnectionString": "Data Source=客戶Oracle伺服器:1521/ORCL;User Id=帳號;Password=密碼;",
    "Provider": "Oracle"
  },
  "SyncSettings": {
    "BatchSize": 100,
    "LogDirectory": "Logs",
    "SyncIntervalMinutes": 60
  }
}
```

### 步驟 3: 測試連線

在客戶環境開啟命令提示字元，執行：

```bash
Sync104ToBpmErp.exe --test-connection
```

確認顯示：
```
BPM 資料庫 (MS-SQL): 連線成功 ✓
ERP 資料庫 (Oracle): 連線成功 ✓
HR API: 連線成功 ✓ (Token 取得成功)
```

### 步驟 4: 測試同步

執行一次測試同步：

```bash
# 互動模式
Sync104ToBpmErp.exe

# 輸入較短的時間範圍測試，例如：
# 開始時間: 2024-01-01 00:00:00
# 截止時間: 2024-01-01 23:59:59
```

檢查 `Logs/` 目錄下的 Log 檔案：
- `SyncLog_YYYYMMDD.txt` - 確認執行流程正常
- `ErrorLog_YYYYMMDD.txt` - 確認沒有錯誤

### 步驟 5: 設定 Windows 工作排程器

1. 開啟「工作排程器」(Task Scheduler)
2. 點擊「建立基本工作...」
3. 設定：
   - **名稱**: HR資料同步排程
   - **觸發程序**: 每天 (或依需求設定)
   - **開始時間**: 例如 凌晨 2:00
   - **重複執行**: 每 1 小時 (依需求調整)
   - **動作**: 啟動程式
   - **程式**: `C:\Sync104ToBpmErp\Sync104ToBpmErp.exe`
   - **引數**: `--sync "{前一天日期} 00:00:00" "{當天日期} 23:59:59"`

#### 引數範例 (每天同步前一天資料):
```
--sync "2024-01-14 00:00:00" "2024-01-14 23:59:59"
```

## 常見問題排除

### 問題 1: 無法連線到 BPM 資料庫

**錯誤訊息**: `無法連線到 BPM 資料庫 (MS-SQL)`

**檢查項目**:
1. 確認 SQL Server 服務正在執行
2. 確認防火牆允許連線
3. 確認連線字串正確 (伺服器名稱、資料庫名稱、帳號密碼)
4. 確認 TCP/IP 協定已啟用

### 問題 2: 無法連線到 ERP 資料庫

**錯誤訊息**: `無法連線到 ERP 資料庫 (Oracle)`

**檢查項目**:
1. 確認 Oracle 服務正在執行
2. 確認 TNS 設定正確 (或使用完整連線字串)
3. 確認防火牆允許連線 (Port 1521)
4. 確認帳號密碼正確

### 問題 3: HR API 連線失敗

**錯誤訊息**: `取得 Access Token 失敗`

**檢查項目**:
1. 確認 API Base URL 正確
2. 確認 API Key 正確且未過期
3. 確認網路可連線到 HR API 伺服器
4. 檢查是否需要 Proxy 設定

### 問題 4: 同步時部分資料失敗

**檢查方式**:
1. 查看 `Logs/ErrorLog_YYYYMMDD.txt`
2. 確認失敗的資料 Key 和錯誤訊息
3. 常見原因：
   - 資料格式不符 (例如字串太長)
   - 必填欄位為空
   - 外鍵約束失敗

## 日常維護

### 檢查同步狀態
定期檢查 `Logs/SyncLog_YYYYMMDD.txt`：
- 確認每天/每小時都有執行記錄
- 確認沒有異常錯誤

### Log 檔案管理
Log 檔案會自動依日期分割，建議：
- 每月清理一次超過 30 天的舊 Log
- 或設定排程自動清理

### 資料庫維護
定期檢查同步資料表：
```sql
-- BPM (MS-SQL)
SELECT COUNT(*) FROM BPM_EMPLOYEE;
SELECT COUNT(*) FROM BPM_DEPARTMENT;
SELECT MAX(SYNC_TIME) FROM BPM_EMPLOYEE;

-- ERP (Oracle)
SELECT COUNT(*) FROM ERP_EMPLOYEE;
SELECT COUNT(*) FROM ERP_DEPARTMENT;
SELECT MAX(SYNC_TIME) FROM ERP_EMPLOYEE;
```
